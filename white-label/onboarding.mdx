Siga os passos abaixo para configurar sua plataforma **White Label da OmniTech IA**:

## 1. Ativar o White Label

1. Contrate o plano White Label
2. Aguarde a confirmação de ativação
3. Acesse o seu dashboard

## 2. Acessar o Painel Administrativo

1. No dashboard, localize o botão **Painel Administrativo**
   <Frame>
     <img
       src="/resources/white-label-button.png"
       alt="Localização do botão do Painel Administrativo"
     />
   </Frame>
2. Clique para acessar as configurações administrativas

## 3. Configurar as Definições da Plataforma

No **Painel Administrativo**, você poderá personalizar completamente sua plataforma.  
Cada seção controla um aspecto específico da identidade, operação e experiência do usuário.

---

### Configuração de Identidade Visual

<Frame>
  <img src="/resources/white-label-brand-identity.png" alt="Configurações de Identidade Visual" />
</Frame>

**Configuração Completa de Branding:**
- **Nome da Plataforma:** Nome exibido em toda a interface e nos e-mails
- **Logo Principal:** Logotipo usado no dashboard e na interface (modo claro)
- **Logo para Modo Escuro:** Logotipo alternativo para tema escuro
- **Imagem da Página de Autenticação:** Imagem personalizada para login/cadastro (modo claro)
- **Imagem da Página de Autenticação (Escuro):** Versão para modo escuro
- **Favicon:** Ícone exibido no navegador (.ico ou .png)
- **Plano de Fundo do Cabeçalho:** Imagem de fundo da página de login (modo claro)
- **Plano de Fundo do Cabeçalho (Escuro):** Versão para modo escuro
- **Altura do Logo:** Altura personalizada do logo em pixels (padrão: 35px)
- **Título da Página:** Texto principal da página de autenticação
- **Descrição da Página:** Subtítulo da página de autenticação
- **Título SEO:** Título exibido no navegador e motores de busca
- **Descrição SEO:** Meta description para buscadores
- **Palavras-chave SEO:** Palavras-chave para otimização de busca

**Requisitos de Imagem:**
- Logos: PNG ou SVG (fundo transparente recomendado)
- Imagens de Autenticação: JPEG/PNG/WebP em alta resolução (1920×1080 recomendado)
- Favicon: 32×32 ou 16×16 pixels
- Fundos de Cabeçalho: mínimo 1920×1080

### Configuração de Domínio Personalizado

<Frame>
  <img src="/resources/white-label-custom-domain.png" alt="Configuração de Domínio Personalizado" />
</Frame>

**Configuração de Domínio:**
- **Domínio Personalizado:** URL da sua plataforma (ex.: app.suaempresa.com)
- **Certificado SSL:** HTTPS automático para segurança
- **Suporte a Subdomínios:** app, platform, ai, phone, entre outros

**Requisitos de DNS:**
- **Tipo de Registro:** A (não usar CNAME)
- **Host/Nome:** Subdomínio escolhido
- **Valor:** Endereço IP informado na plataforma

**Requisito Crítico de SSL:**
- **Adicionar o DNS sem proxy para garantir o funcionamento do SSL**

**Solução de Problemas Comuns:**
- O SSL é gerado automaticamente após DNS correto
- Em caso de falha, verifique se não há proxy ativo
- Usuários do Cloudflare devem desativar o proxy no registro DNS
- O suporte pode regenerar o SSL se necessário
- Aguarde até 24–48h para propagação completa

<Frame>

**Configuração de Cores:**
- **Cor Primária:** Cor principal da marca (botões, links, destaques)

**Boas Práticas:**
- Garanta contraste adequado (WCAG 2.1 AA)
- Teste em modo claro e escuro
- Mantenha consistência visual
- Considere acessibilidade para daltônicos

### Configuração de URLs do Site

<Frame>
  <img src="/resources/white-label-website-urls.png" alt="Configuração de URLs" />
</Frame>

**Links Essenciais:**
- **URL dos Termos de Uso:** Exibido no cadastro de usuários
- **URL da Documentação:** Link para sua documentação personalizada

**Requisitos:**
- URLs devem usar HTTPS
- Preferencialmente no seu domínio
- O link de termos aparece no registro
- A URL da documentação substitui os links padrão de ajuda

**Exemplos:**
- Termos: `https://suaempresa.com/termos`
- Docs: `https://docs.suaempresa.com`

### Integração com Login Social

<Frame>
  <img src="/resources/white-label-social-logins.png" alt="Login Social" />
</Frame>

**Controle de Login Social:**
- Ativação geral de logins sociais
- Integração com Google OAuth
- Integração com LinkedIn OAuth

**Configuração Google OAuth:**
- Client ID e Client Secret
- Redirect URL: `https://seudominio.com/login/google/callback`

**Configuração LinkedIn OAuth:**
- Client ID e Client Secret
- Redirect URL: `https://seudominio.com/login/linkedin-openid/callback`

**Observações Importantes:**
- Desativar logins sociais remove credenciais
- Teste antes de ir para produção
- Usuários ainda podem usar login tradicional
